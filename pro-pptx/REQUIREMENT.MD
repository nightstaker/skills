# Pro-PPTX 项目需求文档 (Production-Ready v2.0)

## 1. 背景与目标 (Background & Goals)

### 1.1 背景
当前主流 PPT 工具过度追求视觉艺术性，但在企业生产力场景中，组织通常有严苛的排版合规性要求（如特定的颜色层级、固定的对齐方式、标准化的栏目位置）。例如：
* **颜色优先级：** 红色 > 黑色 > 灰色。
* **字体优先级：** 微软雅黑 > Arail。
* **结构约束：** 每一页幻灯片右上角必须包含二级类目名称。

### 1.2 核心目标
构建一个生产级 PPT 技能，支持以下核心场景：
1.  **新建：** 基于选定模板从零生成 PPT。
2.  **更新：** 识别现有 PPT 样式并进行内容迭代，支持基于 **Slide ID** 的部分更新，确保用户手动微调后的内容不被覆盖。
3.  **转换：** 将现有 PPT 内容无损迁移至新的目标模板。
4.  **数据联动：** 自动将外部数据（Excel/CSV）转化为符合模板风格的动态图表。

## 2. 模板定义 (Template Definition)

每个模板由一个独立文件夹构成，包含 `template.md`（核心逻辑）和 `template.pptx`（母版参照）。

### 2.1 template.md 结构扩展
```markdown
# Description
描述模板适用场景、行业背景及总体风格限制。

# Visual Constraints
- Color Palette: 明确十六进制色值及使用权重。
- Typography: 规定标题、正文、标注的字体族、字号、颜色及行间距。
- Logo & Assets: 标准 Logo 位置及边距要求。

# Asset Management (NEW)
- Icon Engine: 自动将关键词转换为风格统一的矢量图标（SVG），并适配模板色值。
- Compliance: 自动标注图片来源或进行 CC 协议合规性过滤。

# Structure & Layout
- Slide Master Mapping: 建立内容类型与 template.pptx 中母版页（Layouts）的映射关系。
- Elements: 规定特定元素（页码、二级目录、水印）的绝对坐标位置。
```

## 3. 核心功能模块 (Function Modules)

### 3.1 智能解析器 (PPTX Parser)
* **a. 样式识别与比对：** 提取现有 PPT 样式生成临时模板。计算其与标准库模板的相似度：
    * **相似度 < 80%：** 沿用临时模板，保持原有风格。
    * **相似度 ≥ 80%：** 自动纠偏，将其强制对齐到最接近的标准模板。
* **b. 内容结构提取：** 解析文本、表格及图表，建立“章节-页面-元素”的树状逻辑模型。

### 3.2 内容增强器 (Content Extender)
* **a. 结构化处理：** 将输入内容转换为树形结构（Tree Format），符合模板要求的逻辑层级。
* **b. 信息扩充与推理：** 针对关键节点生成搜索查询，调用外部搜索进行深度思考：
    * **逻辑严密性：** 检查新旧内容是否冲突。
    * **事实核查：** 确保引用数据准确，并生成信息来源标注。
* **c. 协作预览 (Draft Mode)：** 在正式渲染前，先输出 Markdown 格式的“逻辑草图”，由用户确认后再生成文件。

### 3.3 样式设计器 (PPTX Writer)
* **a. 动态排版设计：** 根据内容密度自动选择母版布局。
    * **填充原则：** 优先保证视觉饱满（Fill the entire page），避免大面积空白。
    * **溢出处理：** 若单页内容过长，自动执行“智能拆分”或“提炼摘要”。
* **b. 自动化渲染：** 调用底层的 PPT 渲染引擎生成 `.pptx` 原生可编辑文件。

### 3.4 质量校验器 (Linter & QA)
* **样式核查：** 检查字体一致性、标题位置坐标（防止翻页抖动）及颜色合规性。
* **合规与辅助：** 自动为图片生成 `Alt Text`（无障碍支持），检查文字溢出及拼写错误。

## 4. 生产级技术指标 (Technical Specifications)

| 维度 | 要求说明 |
| :--- | :--- |
| **元素属性** | 所有文本和图形必须是**可编辑状态**，外部图片及用户输入图片例外且优先引用。 |
| **渲染引擎** | 必须支持高保真导出，确保在 Microsoft PowerPoint 和 Keynote 中无乱码。 |
| **增量更新** | 必须支持 Slide ID 锁定，确保在多次迭代中保留用户对手动微调页面的所有权。 |
| **多端适配** | 导出时支持**字体子集嵌入**，并同步导出带书签的 PDF 和长图。 |
