
# Pro-PPTX 项目需求文档 (Production-Ready v3.0)

## 1. 背景与目标 (Background & Goals)

### 1.1 背景
企业级 PPT 核心价值在于**高效决策**与**专业合规**。当前工具多偏向视觉创作，但在生产力场景中，组织对排版一致性、信息密度及数据严谨性有严苛要求。
* **高密度原则**：拒绝低效排版，追求单页内“背景-数据-结论”的完整逻辑。
* **视觉增强**：强制通过高质量外部图片与图标辅助表达，提升专业度。

### 1.2 核心目标
1.  **新建**：基于模板从零生成逻辑严密、视觉饱满的 PPT。
2.  **更新**：支持基于 **Slide ID** 的增量更新，确保 AI 迭代时保留用户对手动微调页面的所有权。
3.  **转换**：将现有内容无损迁移至新模板，相似度 ≥80% 时自动执行样式纠偏。
4.  **数据联动**：自动将外部数据（Excel/CSV）转化为符合模板风格的动态图表。

## 2. 模板定义 (Template Definition)

每个模板由独立文件夹构成，包含 `template.md`（逻辑协议）和 `template.pptx`（母版参照）。

### 2.1 template.md 结构规范
```markdown
# Description
描述模板适用场景、行业背景及总体风格限制。

# Visual Constraints
- Color Palette: 十六进制色值及权重（如：Red > Black > Gray）。
- Typography: 规定各级标题与正文的字体族、字号及颜色。
- Asset Management: 自动匹配模板色值的 Icon 引擎；外部图片需通过合规性过滤。

# Content Constraints (NEW)
- Visual Mandatory: 每一页核心内容必须关联至少一张外部图片或图表，禁止纯文字页面。
- Information Density: 单页字数（含标注）原则上不低于 150 字，涵盖 3-5 个深度逻辑要点。
- Visual-Text Ratio: 视觉元素占比不低于 40%，且文字需围绕图片进行深度解说。

# Structure & Layout
- Slide Master Mapping: 建立内容类型与母版页（Layouts）的绝对映射关系。
- Elements: 定义页码、二级目录、水印等固定组件的绝对坐标位置。
```

## 3. 核心功能模块 (Function Modules)

### 3.1 智能解析器 (PPTX Parser)
* **样式纠偏**：提取现有 PPT 样式并计算与标准模板的相似度，低于 80% 则沿用原风格，高于 80% 则强制对齐标准。
* **结构提取**：将文本、表格及图表转化为“章节-页面-元素”的树状模型。
### 3.2 内容增强器 (Content Extender)
* **逻辑稠密化**：针对简略输入，AI 必须通过推理引擎自动补充行业背景、案例或数据预测，确保信息量饱满。
* **视觉触发逻辑**：为每页内容自动生成具体的“意象描述”并执行外部图片搜索。
* **协作预览 (Draft Mode)**：在正式渲染前输出 Markdown 格式的“逻辑草图”，由用户确认后再生成文件。

### 3.3 样式设计器 (PPTX Writer)
* **饱和填充算法**：优先采用“左图右文”或“多栏阵列”布局，杜绝大面积留白。
* **智能溢出策略**：内容过长时优先执行“层级折叠”或“提炼摘要”，而非直接删减或简单分拆。

### 3.4 质量校验器 (Linter & QA)
* **密度检查**：自动识别并打回“低信息量页面”，强制要求增加论据或视觉组件。
* **样式与合规**：检查字体一致性、标题坐标抖动；自动为图片生成 `Alt Text`（无障碍支持）。

## 4. 生产级技术指标 (Technical Specifications)

| 维度 | 要求说明 |
| :--- | :--- |
| **元素属性** | 所有文本和图形必须是**可编辑状态**，外部图片及用户输入图片例外且优先引用。 |
| **渲染引擎** | 高保真导出，确保在 Microsoft PowerPoint 和 Keynote 中排版无损。 |
| **增量更新** | 必须支持 Slide ID 锁定，确保手动微调内容不被 AI 覆盖。 |
| **多端适配** | 导出时支持**字体子集嵌入**，同步产出带书签的 PDF 和长图。 |
| **性能响应** | 高密度内容生成（含多节点搜索。 |
